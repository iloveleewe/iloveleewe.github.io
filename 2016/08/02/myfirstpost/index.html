瞎折腾之Github+Hexo+Next建独立博客

前段时间入手了DO的虚拟VPS，主要是为了当做梯子用，毕竟国内的网络环境堪忧啊，加之需要时不时的查找国外的文献和看一些时事美剧，所以毫不犹豫的入手了！在整个搭建VPS的过程中，意外收获就是Paypal，这样的工具或者说这样的公司是值得尊敬的：非常奈斯的客服服务，非常人性化的促销方式，非常好的企业文化和非常好国内人缘！同时，知道了VI和VIM工具的使用，想想也是非常的反人类啊，各种指令忙个不停；Ubuntu照样的那样让我着迷，只是奈何不了你！

心血来潮，想自己建一个属于自己的博客，于是开始了Github+Hexo+Next之路，注定一条“平凡之路”….

安装准备软件

  • 1.Node.js
  • 2.Git

严格意义上算起来，这是我第一次接触GitHub，难免有很多不熟悉的地方！首先是在官网上去注册。注册的username和邮箱，邮箱十分重要，GitHub上很多通知都是通过邮箱的。然后就是配置和使用Github，其中包括：新建自己博客的仓库，下载Git客户端，下载Hexo并克隆主题等。

传送门：

  • Github官网：http://www.github.com/
  • 使用Github Pages建独立博客：http://beiyuu.com/github-pages

独立域名购买和解析

接下来就是去“购买”域名了，学生党穷，没钱慌啊，果断入手免费域名来练手。悉知原来大名鼎鼎的Tk域名代理商被收购了，被一家叫Feeenom的公司收购，在Freenom申请TK,ML,GA,CF免费顶级域名有两个关键点：一是用DL，否则在创建Freenom账号时不能成功；二是用自己的DNS服务器，或者使用不常用的DNS服务器，否则直接使用DNSPod的DNS服务器会提示不符合要求。做到这两点基本上就可以成功注册到域名了。下面是具体步骤：

一、Freenom注册域名和创建账号

二、Freenom快速注册域名

三、Freenom使用自己的DNS服务器

四、Freenom免费域名使用DNSPod解析域名

详细教程见：https://www.freehao123.com/freenom-tk-ml-ga-cf/

将独立域名与GitHub Pages的空间绑定

GitHub Pages的设置

方法一：在Repository的根目录下面，新建一个名为CNAME的文本文件，里面写入你要绑定的域名，比如cnfeat.com，加不加www其实不太重要。

方法二：到我的Github仓库，点击右下角的「Download ZIP」，下载源文件，解压，找到CNAME文件，用记事本打开，将cnfeat.com修改成你的域名，加不加www其实不太重要。放进Hexo\source目录下，用hexo命令提交上去。

hexo d -g
